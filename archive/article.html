Guía Completa para Construir Aplicaciones de IA con Amazon Bedrock

¡Hola a todos! Hoy exploraremos Amazon Bedrock, el servicio de AWS que nos permite construir aplicaciones de IA generativa de manera sencilla. A lo largo de esta guía, verás cómo empezar, qué características ofrece Bedrock y te llevaré a través de un ejemplo práctico paso a paso. Al final, tendrás una comprensión clara de cómo utilizar este servicio y estarás listo para implementarlo en tus proyectos.

Introducción a Amazon Bedrock

Amazon Bedrock es un servicio totalmente gestionado que facilita el acceso a modelos de inteligencia artificial, incluyendo los que Amazon ha desarrollado y los de terceros. Esto te permite crear aplicaciones innovadoras sin necesidad de preocuparte por la infraestructura subyacente. Puedes usar Bedrock para generar texto, crear imágenes, e incluso desarrollar modelos personalizados.

En esta guía, nos enfocaremos en cómo usar Bedrock para crear un generador de texto alternativo para imágenes, una aplicación que es útil para mejorar la accesibilidad.

Requisitos Previos

Antes de empezar, asegúrate de tener lo siguiente:

  • Una cuenta de AWS activa.
  • Acceso a Amazon Bedrock (puedes solicitar acceso en la consola de AWS).
  • Conocimientos básicos de JavaScript y Node.js.

Paso a Paso: Generador de Texto Alternativo para Imágenes

1. Configuración del Entorno

Primero, vamos a crear un nuevo proyecto de Node.js. Abre tu terminal y sigue estos pasos:

mkdir img-alt-gen
cd img-alt-gen
npm init -y

Luego, instalaremos el SDK de Bedrock:

npm install @aws-sdk/client-bedrock-runtime

2. Configuración de AWS IAM

Para interactuar con Bedrock, necesitas configurar las políticas de IAM. Ve a la consola de IAM en AWS y crea una política que permita el acceso total a Bedrock. Aquí tienes un ejemplo de política JSON:

{
 "Version": "2012-10-17",
 "Statement": [
   {
     "Sid": "BedrockFullAccess",
     "Effect": "Allow",
     "Action": ["bedrock:*"],
     "Resource": "*"
   }
 ]
}

Asigna esta política a tu usuario o rol de IAM.

3. Creación del Archivo del Script

Crea un archivo index.js y añade el siguiente código para interactuar con Bedrock:

import {
  BedrockRuntimeClient,
  InvokeModelCommand,
} from "@aws-sdk/client-bedrock-runtime";

const brockClient = new BedrockRuntimeClient();
const prompt = "Describe la imagen (objetos y texto) para el texto alternativo. Escribe solo el texto en una frase.";

const modelInput = {
  messages: [
    {
      role: "user",
      content: [
        {
          text: prompt,
        },
      ],
    },
  ],
};

const modelCmd = new InvokeModelCommand({
  modelId: "amazon.nova-lite-v1:0",
  body: JSON.stringify(modelInput),
});

const res = await brockClient.send(modelCmd);
const jsonRes = Buffer.from(res.body).toString("utf8");
console.dir(JSON.parse(jsonRes), { depth: null });

4. Enviar Imágenes a Bedrock

Para enviar imágenes, primero lee la imagen como base64. Modificaremos el script para incluir esta funcionalidad:

import { readFile } from "fs/promises";

const base64Img = await readFile("./path-to-your-image.jpg", "base64");
const prompt = "Describe la imagen (objetos y texto) para el texto alternativo. Escribe solo el texto en una frase.";

const modelInput = {
  messages: [
    {
      role: "user",
      content: [
        {
          image: {
            format: "jpeg",
            source: { bytes: base64Img },
          },
        },
        {
          text: prompt,
        },
      ],
    },
  ],
};

5. Ejecución del Script

Ejecuta tu script con el siguiente comando en la terminal:

node index.js

Deberías ver un resultado que describe la imagen que enviaste.

Mejores Prácticas y Errores Comunes

Mejores Prácticas

  • Optimización de Tokens: Asegúrate de usar un número prudente de tokens en tus llamadas para optimizar costos.
  • Revisión de Políticas de IAM: Revisa siempre las políticas de acceso para asegurar que no otorgas más permisos de los necesarios.

Errores Comunes

  • Problemas de Permisos: Si recibes errores de acceso denegado, revisa tus políticas de IAM.
  • Formateo de Datos: Asegúrate de que los datos que envíes a Bedrock estén en el formato correcto.
